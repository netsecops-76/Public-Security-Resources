Qualys Cloud Agent Log Viewer

A browser-based tool for parsing and analyzing Qualys Cloud Agent logs. No server, no install — just open the HTML file and drop your logs.

Getting Started

Drag and drop log files onto the upload zone, or use the Browse Files / Upload Folder buttons. The viewer auto-detects the log type and compression format.

Toolbar Controls

  • Search — Real-time partial-match search across all log messages. Multiple words are matched independently (AND logic). Matches are highlighted in yellow.
  • Level Filters (All / Trace / Debug / Info / Warn / Error) — Show only log lines at the selected severity level. Error filter also includes PowerShell error blocks.
  • Redact — Toggle sensitive data masking (on by default). Redacts CustomerIDs, AgentIDs, IPs, MAC addresses, serial numbers, HMAC tokens, and UUIDs in URLs. Redaction is display-only — search still works on original values.
  • Newest First / Oldest First — Toggle sort order. Default is newest first since troubleshooting usually starts from the latest entries.
  • Select — Enter line selection mode. Checkboxes appear next to each log line. Pick individual lines, then use Export Selected to download a plain-text file with host details and each selected line separated by divider lines. The button changes to Cancel to exit selection mode. Use Select All / Deselect All for bulk operations. Exported files are named hostname_logname_YYYYMMDDHHMMSS.txt.

Smart Grouping

Log entries are grouped semantically based on the detected log type:

  • Cloud Agent — CAPI cycles, scan events, module lifecycle, agent startup/shutdown
  • CEP — Component lifecycle, manifest execution, polling with retries
  • Health Check — Individual check sections (CPU, RAM, disk, connectivity, proxy)
  • Scan — Manifest processing blocks, command executions, skipped OS mismatches (batched)
  • Other — Time-gap-based grouping for unrecognized formats

Click a group header to collapse or expand it.

JSON Payloads

CAPI request and response payloads are detected automatically. Click the {JSON} button on a log line to expand a syntax-highlighted view. Keys appear in blue, strings in green, numbers in orange, and booleans in purple. Sensitive fields are redacted when redaction is enabled.

Multi-File Tabs

When loading multiple files, each gets its own tab. The viewer defaults to qualys-cloud-agent.log (Linux) or Log.txt (Windows) when available. Arrow buttons appear at the edges of the tab bar when there are more tabs than fit on screen. Tab badges indicate status:

  • 5 Error count (red) — files with errors appear first
  • 3 Warning count (yellow)
  • OK No errors or warnings (green)

Compression & Archives

The viewer handles multiple compression formats transparently:

  • Gzip — Inline fflate decoder with truncation-tolerant streaming. Recovers all decompressed data up to the truncation point in agent-truncated REMOTELOG bundles.
  • Deflate — Browser-native (used for ZIP internal decompression)
  • LZMA — Inline JavaScript decoder for Linux REMOTELOG bundles
  • 7z / LZMA2 — Loaded from CDN on first use (requires internet). Falls back to manual extraction instructions if offline.

Double-compressed archives (LZMA wrapping gzip, or 7z inside ZIP) are detected and handled automatically. Truncated archives (common with Linux REMOTELOG bundles that hit agent collection size limits) are handled gracefully — partial data is recovered rather than discarded.

Version Check & Auto-Update

On page load, the viewer checks for updates by fetching a version file from the GitHub repository. If a newer version is available, a yellow badge appears in the header showing how many versions behind you are (e.g., N-1 = one version behind, N-3 = three versions behind).

Clicking Update opens a dropdown with two options:

  • Download Now — Fetches the latest version from GitHub and downloads it with a versioned filename (e.g., qualys-log-viewer-1.0.9.html). Open the new file to start using the updated version. You can delete the old version afterward.
  • Go to GitHub — Opens the repository in a new tab to browse or download manually.

The dropdown also shows where your current copy is located so you know exactly where to save the update.

Update check unavailable? SSL-inspecting proxies such as Zscaler, Netskope, or corporate firewalls can block outbound requests to raw.githubusercontent.com, preventing the version check from completing. When this happens, the badge will show "Update check unavailable" instead of a version status. This does not affect any viewer functionality — all log parsing and analysis works fully offline.

To manually check for updates, visit the repository directly:

github.com/netsecops-76/Public-Security-Resources → Qualys-Cloud-Agent-Log-Viewer branch

Browser Requirements

Chrome 105+, Edge 105+, Firefox 113+, or Safari 16.4+. The 7z decoder requires network access to jsDelivr CDN.

Tips

  • Use the Error filter to quickly find problems in large log files
  • Turn off Redact when you need to see actual IDs for cross-referencing
  • Switch to Oldest First when tracing a sequence of events chronologically
  • Drag an entire REMOTELOG folder or ZIP directly onto the viewer — it handles all the nested extraction
  • For very large files, groups beyond the first 5000 lines are collapsed by default — click to expand, or use Load More at the bottom
