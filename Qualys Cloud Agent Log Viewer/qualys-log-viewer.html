Qualys Cloud Agent Log Viewer

A browser-based tool for parsing and analyzing Qualys Cloud Agent logs. No server, no install — just open the HTML file and drop your logs.

Getting Started

Drag and drop log files onto the upload zone, or use the Browse Files / Upload Folder buttons. The viewer auto-detects the log type and compression format.

Toolbar Controls

  • Search — Real-time full-text search across all log messages. Matches are highlighted in yellow.
  • Level Filters (All / Trace / Debug / Info / Warn / Error) — Show only log lines at the selected severity level. Error filter also includes PowerShell error blocks.
  • Redact — Toggle sensitive data masking (on by default). Redacts CustomerIDs, AgentIDs, IPs, MAC addresses, serial numbers, HMAC tokens, and UUIDs in URLs. Redaction is display-only — search still works on original values.
  • Newest First / Oldest First — Toggle sort order. Default is newest first since troubleshooting usually starts from the latest entries.
  • New File — Reset and load different files.

Smart Grouping

Log entries are grouped semantically based on the detected log type:

  • Cloud Agent — CAPI cycles, scan events, module lifecycle, agent startup/shutdown
  • CEP — Component lifecycle, manifest execution, polling with retries
  • Health Check — Individual check sections (CPU, RAM, disk, connectivity, proxy)
  • Scan — Manifest processing blocks, command executions, skipped OS mismatches (batched)
  • Other — Time-gap-based grouping for unrecognized formats

Click a group header to collapse or expand it.

JSON Payloads

CAPI request and response payloads are detected automatically. Click the {JSON} button on a log line to expand a syntax-highlighted view. Keys appear in blue, strings in green, numbers in orange, and booleans in purple. Sensitive fields are redacted when redaction is enabled.

Multi-File Tabs

When loading multiple files, each gets its own tab. Tab badges indicate status:

  • 5 Error count (red) — files with errors appear first
  • 3 Warning count (yellow)
  • OK No errors or warnings (green)

Compression & Archives

The viewer handles multiple compression formats transparently:

  • Gzip & Deflate — Browser-native, no dependencies
  • LZMA — Inline JavaScript decoder for Linux REMOTELOG bundles
  • 7z / LZMA2 — Loaded from CDN on first use (requires internet). Falls back to manual extraction instructions if offline.

Double-compressed archives (LZMA wrapping gzip, or 7z inside ZIP) are detected and handled automatically.

Browser Requirements

Chrome 105+, Edge 105+, Firefox 113+, or Safari 16.4+. The 7z decoder requires network access to jsDelivr CDN.

Tips

  • Use the Error filter to quickly find problems in large log files
  • Turn off Redact when you need to see actual IDs for cross-referencing
  • Switch to Oldest First when tracing a sequence of events chronologically
  • Drag an entire REMOTELOG folder or ZIP directly onto the viewer — it handles all the nested extraction
  • For very large files, groups beyond the first 5000 lines are collapsed by default — click to expand, or use Load More at the bottom
